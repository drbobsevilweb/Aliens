═══════════════════════════════════════════════════════════════════════════════
KEY TECHNICAL DECISIONS
═══════════════════════════════════════════════════════════════════════════════

DATA FORMAT (BACKWARD-COMPATIBLE)
  Current: { x, y, width, height }
  Enhanced: { x, y, width, height, _style: {...}, breakpoints: {...} }
  
  Fallback: If _style missing, use hardcoded HUD_COLORS from HUD.js
  Migration: No schema version needed; optional fields handled gracefully

COLOR PICKER IMPLEMENTATION
  Use native <input type="color"> (browser built-in)
  Parse: #rrggbb ↔ 0xrrggbb conversion via parseInt(color.replace('#', ''), 16)
  Preview: Update on change event, not on blur (immediate feedback)

FONT RENDERING
  System fonts only (Share Tech Mono, IBM Plex Mono, Georgia, sans-serif)
  No web font loading (keep editor performant)
  Font prop: "fontSize weight family" for ctx.font = "..."

BREAKPOINT STRUCTURE
  Each panel tracks positions separately per breakpoint
  Use nested: panel.breakpoints[bpKey] = { x, y, width, height, _style }
  Active breakpoint: selectedPanel uses breakpoints[activeBreakpoint]

ALIGNMENT ALGORITHM
  Distribute: Calculate average spacing = (rightmost - leftmost) / (count - 1)
  Snap: Math.round(value / gridSize) * gridSize
